<h2>Project Overview</h2>
This program was developed as a required group assignment for the Object-Oriented Programming 2 course at the University of South-Eastern Norway. Completing this assignment was a mandatory prerequisite to qualify for the final exam. While our submission was approved, the program is still in an early stage and lacks polish and final features requested in the specification.

<h3>Features & Functionality</h3>
    <ul>
        <li><strong>Simulation of a Restaurant:</strong> Simulates a running restaurant with customers ordering meals and chefs preparing them.</li>
        <li><strong>Concurrency Using Threads:</strong> Uses separate threads for chefs and customers to simulate independent actions.</li>
        <li><strong>Time and State Simulation:</strong> Handles delays using thread sleep states to represent preparation times, resting periods, and waiting times.</li>
        <li><strong>Customer Satisfaction Tracking:</strong> Features a counter for "happy" and "unhappy" customers based on the total waiting time for their orders.</li>
        <li><strong>Data Logging:</strong> Instead of a database, the application logs data directly to a text file.</li>
    </ul>

<h3>Technology Stack</h3>
    <ul>
        <li><strong>Language:</strong> Java</li>
        <li><strong>Team Size:</strong> 5 members</li>
    </ul>
